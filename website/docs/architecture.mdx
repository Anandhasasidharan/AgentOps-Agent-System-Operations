## Shared Infrastructure

### Database
All services share a single **PostgreSQL 16** database. Each service uses its own table namespace. SQLAlchemy 2.0 async sessions with asyncpg driver.

### Authentication
API key format: `slug:random-hex-token` (e.g. `acme-corp:a1b2c3...`). Sent as `X-API-Key` header, parsed by splitting on `:`. Hash stored in `Tenant.api_key_hash`. Key returned once on creation.

### Events (NATS)
The `agentops-events` package provides:
- **6 NATS topics** for inter-service events
- `publish_event()` with retry logic
- No-op publisher when NATS is unavailable
- Topic constants imported from `agentops_events` package root

### Rate Limiting
Per-tenant in-memory **sliding window** (60s) in `agentops_core.rate_limiter`. Configurable via `RATE_LIMIT_RPM` env var (default 60 req/min/tenant). Applied to CB, Chaos, SLO. `/health` and `/metrics` bypass the limiter.

### Telemetry
OpenTelemetry OTLP emission is **fire-and-forget HTTP POST** with OTLP JSON format. Prometheus scrapes all 5 services on `/metrics` (ports 8000-8004).

## Lifecycle

```
DEVELOPMENT ──→ Chaos Toolkit validates agent resilience
                    │
                    ▼
STAGING ──→ Circuit Breaker enforces policies on real tool calls
                    │
                    ▼
PRODUCTION ──→ SLO Platform monitors SLIs, alerts on breaches
                    │
                    ▼
         Compliance reports (OWASP, EU AI Act)
```
